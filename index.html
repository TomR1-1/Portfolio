Sandy (2023) - Front Wing

CFD simulation of the front wing in the context of the full car
CNC routering the mould for the second
Fitting the main element mould into the environmental chamber to cure
Manufactured front wing endplates
Front wing manufactured and mounted on the car
< >

I made the 2023 front wing for Formula UBC Racing on a tight timeline of three months. I used SolidWorks to design the wing, using surfaces for all the wing elements, and solid bodies for the simpler shapes. Star-CCM+ was used to simulate its performance with the k-w SST turbulence model at a wall y+ < 5. HMSWorks was used to toolpath the mould for the wings, and CNC-ed out of MDF. Lastly, pre-preg carbon fibre was laid up on the prepped mould surface, cured, then assembled.

Unfortunately, the member who I assigned the task to disappeared midway through the season, which left me, the aerodynamics team lead, to engineer this crucial component within the tight timeline. In 2022, the team ran a triple element front wing inboard of the tyres and ran dual elements in front of the tyres (as per rules restrictions). However, for the 2023 season, we ran the first ever monocoque chassis, which resulted in me not being able to reuse the 2022 front wing.

The first step for me was to read internal documentation of previous front wings. I knew to manufacture a similar wing to the 2022 year would be not possible given the tight timeline, which meant I had to read about previous geometries of complete dual element front wings.

The front wing had to meet two main objectives; to be easy to remove and install, and to be lighter than the previous year. To account for serviceability, I took advantage of the flat portion of the monocoque chassis to install the front wing with four, easy to access bolts. I then designed the internals of the wing to consist of completely carbon fibre sandwich panel stringers, lighter than the spar and aluminum ribs previously implemented. The wing has a CD of 0.15 and a CL of 0.60.

Website Portfolio (WIP)

Testing mobile display & responsivenes
Initial image sizing problem
Working on editing this modal
< >

This website is coded in HTML, CSS, and JavaScript with a little bit of jQuery. No ChatGPT or AI was used in creating this website! Prior to making this website, I had some web dev experience, making a photo storage system using the Django framework, but I primarily worked on the backend in Python. Furthermore, this is the first time I made this website mobile responsive.

The main challenge of this website was setting up the “My Projects” section. First, I had several issues regarding image sizing. It upset me how differently sized images looked uneven. I ended up encountering a solution to this problem after some time away on a social media app. It consisted of using aspect ratios, a maximum width, and making sure the image fit the container it was in while keeping the same aspect ratio (“object-fill: contain;”). Once I found this solution, it became a game of styling the page to my liking – this part is quite difficult itself without tools such as Figma.

My favourite part about my website is the pop-up modal that you are reading this text from. It required a lot of research and tuning, but it turned out great!

A key limitation I encountered is that this website needed to be static to host it on GitHub. This meant that despite having experience using the Django framework, I was not able to design a portfolio with it since it is a dynamic website. In the future I will consider this option.

Surfboard Fin RTM Mould

Problematic BPA meshed point cloud in SolidWorks
Pseudo OOP style code - to be turned into a class soon
Point cloud generated by open3d with NACA 4321 configuration
< >

The goal of this project is to eventually create a Resin Transfer Moulding mould of a surfboard fin with a variety of NACA aerofoil shapes to be used in the UBC Manufacturing Engineering program’s second year course MANU 230. I am working with a group of six volunteers under a professor.

The requirement for this project is to code a script in Python using Open 3D and NumPy libraries, to generate a point cloud and a mesh of the fin variably depending on the MPXX NACA airfoil type a user defines. The mesh is then to be imported into SolidWorks which can be quickly turned into a mould. Unfortunately, as a volunteer project with students who are not experienced with coding, I am forced to take on most of the work.

I used an OOP approach and coded functions that can generates a two-dimensional NACA airfoil based on equations derived from literature using cosine spacing. I then abstracted a function that generates the guiding splines written by another member and produced a point cloud of the fin. I then read up on and tested a variety of meshing algorithms and noticed the Ball Pivot Algorithm provides the best mesh so far.

Currently the mesh has some issues along the guiding splines. I suspect it has something to do with calculating normal of the initially generated point cloud. I am currently reading up on how to calculate the normal values to get a higher quality mesh.
